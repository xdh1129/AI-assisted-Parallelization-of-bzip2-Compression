  These became explicit constraints (documented in src/main.c):
  - Threads clamped to online CPU count (sysconf) ‚ÄĒ beyond that, only
  contention.
  - Queue depth = threads √ó 4, sized for throughput (keep every worker fed +
  reader/writer overlap) rather than to cap RAM, since RAM is abundant.
  - Block size default 900 KB = bzip2's largest window (level¬†9); a single
  block's working set stays within the per-core L2 + shared L3 budget even at
  full thread count.

  Architecture ‚ÄĒ a new src/pipeline.c/.h implementing reader ‚Üí worker pool ‚Üí
  writer:
  - Reader (calling thread) streams fixed-size blocks, assigns increasing IDs,
  applies backpressure (blocks once read_count ‚ąí next_write == qdepth).
  - N worker threads compress blocks independently into standalone .bz2 streams
  ‚ÄĒ the only heavy work, done outside any lock.
  - One writer thread emits blocks in original ID order via a qdepth-sized ring
  reorder buffer. The backpressure guarantees in-flight IDs span a window <
  qdepth, so id % qdepth never collides ‚ÄĒ no per-block allocation table, no
  global locks beyond one mutex + three condvars.
  - --threads 1 keeps a genuine sequential path (no threads created).
  - Empty input still emits one valid empty .bz2 stream.
