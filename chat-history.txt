  What I changed

  Makefile вҖ” added -fopenmp to CFLAGS.

  src/main.c вҖ” turned the sequential compress loop into a parallel one:
  - omp_set_num_threads(opt.threads) so --threads N controls the pool.
  - #pragma omp parallel for schedule(dynamic) over the blocks. Dynamic
  scheduling balances load, since block compression times vary (some blocks
  compress far faster than others).
  - Replaced the OpenMP-illegal break-on-error with a per-iteration if (err) 
  continue; guard and #pragma omp atomic write to set the error flag. The
  post-loop err check is unchanged and runs after the implicit barrier, so
  failures are still reported.

  Why it's safe

  Each iteration writes only its own comp[i] slot вҖ” no shared mutable state.
  input_bytes was already summed outside the loop, and compress_block is
  self-contained (BZ2_bzBuffToBuffCompress is stateless; the lone static is
  read-only). The writer still emits blocks in index order, so output is
  deterministic.
